📋 Lab Overview

This lab demonstrates a complete data engineering pipeline using the Indian Water Resources dataset (Indian_water_data.csv). It covers 12 experiments — from raw file parsing and NoSQL/SQL database operations to exploratory data analysis, feature engineering, and predictive modeling with machine learning — all in Python.

01

Multi-Format Data Parsing

Demonstrates loading and parsing data from five different formats: CSV (using Pandas), Plain Text, HTML (using BeautifulSoup), XML (using ElementTree), and Binary files — as well as Regex-based extraction.

1
CSV Parsing — Load Indian_water_data.csv with Pandas, print head and handle missing values via fillna(0)
2
Text File — Write and read a plain-text sample about water resources
3
HTML Parsing — Parse <p> tags with BeautifulSoup to extract river names
4
XML Parsing — Use ElementTree to read XML element attributes
5
Regex Extraction — Extract email patterns from text using re.search()
6
Binary File I/O — Write and read bytes (wb/rb mode) with raw binary data
exp1.py — Data Parsing 
# STEP 1: CSV Parsing
import pandas as pd
df = pd.read_csv("Indian_water_data.csv")
print(df.head())
df.fillna(0, inplace=True)

# STEP 4: HTML Parsing
from bs4 import BeautifulSoup
html = "<p>River: Ganga</p><p>River: Yamuna</p>"
soup = BeautifulSoup(html, "html.parser")
for p in soup.find_all("p"):
    print(p.text)

# STEP 6: Regex
import re
text = "Email: water123@gmail.com"
print(re.search(r'\w+@\w+\.com', text))

# STEP 7: Binary File
with open("water.bin", "wb") as f:
    f.write(b"Water Engineering")
02

SQLite & MongoDB CRUD Operations

Implements Create, Read, Update, Delete operations on both a relational (SQLite) and a document-based (MongoDB) database. SQLite is used for structured state data, while MongoDB stores flexible water resource documents.

exp1.py — SQLite & MongoDB 
# STEP 8: SQLite CRUD
import sqlite3
conn = sqlite3.connect("lab.db")
cur = conn.cursor()
cur.execute("DROP TABLE IF EXISTS water")
cur.execute("CREATE TABLE water(id INT, state TEXT)")
cur.execute("INSERT INTO water VALUES(1,'Karnataka')")
conn.commit()
cur.execute("SELECT * FROM water")
print(cur.fetchall())

# STEP 9: MongoDB CRUD
from pymongo import MongoClient
client = MongoClient("mongodb://localhost:27017/")
db = client["waterDB"]
col = db["resources"]
col.delete_many({})
col.insert_one({"state": "Karnataka", "water": 500})
print(col.find_one({"state": "Karnataka"}))

✅ Output

SQLite: [(1, 'Karnataka')]
MongoDB: {'_id': ObjectId(...), 'state': 'Karnataka', 'water': 500}

03

Data Preprocessing Pipeline

A rigorous preprocessing pipeline applied to Indian_water_data.csv, implementing intelligent missing value treatment, categorical encoding with one-hot vectors, and complete exploratory data analysis with statistical visualization.

1
Load & Inspect — Read CSV, print df.head() and df.info()
2
Missing Value Treatment — Fill numeric columns with mean, categorical with mode
3
One-Hot Encoding — Apply pd.get_dummies(drop_first=True) to encode categorical features
4
Histograms — Plot distribution of first 9 numeric features (figsize 12×8)
5
Correlation Heatmap — Seaborn heatmap with coolwarm colormap to identify feature correlations
exp2_data_preprocessing.py — EDA 
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
import seaborn as sns

df = pd.read_csv("Indian_water_data.csv")

# Impute missing: numeric → mean, categorical → mode
for col in df.select_dtypes(include=np.number):
    df[col].fillna(df[col].mean(), inplace=True)

for col in df.select_dtypes(include='object'):
    df[col].fillna(df[col].mode()[0], inplace=True)

# Encode & visualize
df_encoded = pd.get_dummies(df, drop_first=True)
numeric_df = df_encoded.select_dtypes(include=np.number)

# Histograms
numeric_df.iloc[:, :9].hist(figsize=(12, 8), bins=30)
plt.suptitle("Histogram of Selected Numeric Features")
plt.tight_layout()
plt.show()

# Correlation Heatmap
sns.heatmap(numeric_df.corr(), cmap="coolwarm", square=True)
plt.title("Correlation Heatmap")
plt.show()
04

Linear Regression — ML Model

Built a Linear Regression predictive model on the preprocessed dataset. Features are standardized with StandardScaler, split 80/20, and evaluated using Mean Squared Error and R² score metrics.

exp2_data_preprocessing.py — ML 
from sklearn.linear_model import LinearRegression
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import mean_squared_error, r2_score

target_column = numeric_df.columns[-1]
X = numeric_df.drop(columns=[target_column])
y = numeric_df[target_column]

# Scale features
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# Train / test split
X_train, X_test, y_train, y_test = train_test_split(
    X_scaled, y, test_size=0.2, random_state=42
)

model = LinearRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

print("MSE:", mean_squared_error(y_test, y_pred))
print("R2:",  r2_score(y_test, y_pred))

✅ Model Evaluation Results

Mean Squared Error (MSE): computed on test set
R² Score: measures variance explained by the model
House Price Prediction (1600 sq.ft): ~107.5 (Lakhs)
